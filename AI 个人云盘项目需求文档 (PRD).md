# AI 个人云盘项目需求文档 (PRD)

## 1. 文档概述

- **项目名称**：AI 个人云盘 (AI Personal Cloud Drive)
- **项目定位**：基于本地硬件存储与手机热点 IPv6 公网能力的轻量级、高带宽、具备 AI 扩展能力的个人私有云盘。
- **当前阶段**：PoC (概念验证) 已跑通，准备进入正式开发与方案优化阶段。

## 2. 背景与痛点分析

- **痛点一（资源开销与速度限制）**：传统第三方网盘（如某网盘）或即时通讯工具（如微信、QQ）在进行文件传输时，存在本地内存占用高、上传/下载速率受限（非会员限速）的问题。
- **痛点二（资源浪费）**：校园网卡或手机套餐内月度公网流量结余较多，未能得到充分利用。
- **核心方案**：以闲置本地磁盘作为存储介质，利用手机热点提供的公网 IPv6 能力进行外网暴露，省去公网服务器与商业网盘的成本，实现不限速的本地文件远程同步。

## 3. 已实现业务流程回顾

1. **域名解析**：域名 `files.balabalashowtime.icu` 已托管至腾讯云 DNS，手动配置的 AAAA 记录指向当前手机热点的公网 IPv6 地址。
2. **基础服务**：本地通过执行 `python -m uploadserver 8000` 启动了基础的 HTTP 上传服务。
3. **服务访问**：外网可通过 `http://files.balabalashowtime.icu:8000` 访问该指定文件夹并进行文件上传与下载。

## 4. 核心功能需求 (MVP 阶段)

### 4.1 文件传输与完整性校验

- **需求描述**：支持多终端（手机、电脑、平板）通过浏览器进行文件的上传与下载。
- **完整性保证**：
  - **哈希校验**：传输前/传输后自动在前端与后端计算文件的 **MD5 / SHA-256 校验码**，比对一致性，确保文件不损坏。
  - **异常处理**：传输失败引发的损坏文件（如临时产生的 `.tmp` 文件）系统需具备自动清理机制，并向用户提示重新上传。

### 4.2 动态域名解析 (DDNS)

- **需求描述**：由于手机热点断开重连或运营商定期刷新，公网 IPv6 地址会发生动态变动，需确保域名能实时指向最新的 IPv6 地址。
- **技术方案**：
  - 使用轻量级 **DDNS 脚本/工具**（如基于 Python 编写的自动化脚本，或部署开源的 `ddns-go`）。
  - 系统定时（如每 3~5 分钟）检测本地热点网卡的公网 IPv6 地址。
  - 当检测到地址变化时，自动调用**腾讯云 DNS API (DNSPod API)**，动态更新 `files.balabalashowtime.icu` 的 AAAA 记录。

### 4.3 安全传输与加密 (HTTPS 升级)

- **需求描述**：将现有的 HTTP 协议升级为 HTTPS 协议，防止传输内容在公网（尤其是在校园网或公共 Wi-Fi 环境中）被监听、窃听或篡改。
- **技术方案**：
  - **证书申请**：使用 **Let's Encrypt / Certbot**（或申请腾讯云免费的 SSL 证书）获取子域名的 HTTPS 证书。
  - **Nginx 反向代理**：在本地部署 Nginx 作为前端网关，负责监听 `443` 端口并配置 SSL 证书，将安全的 HTTPS 请求反向代理至后端的 Python 文件服务。
  - **自动续签**：配置自动化脚本（如通过 Linux Crontab 或 Windows 任务计划），定期自动续签证书。

### 4.4 服务自启动与运维日志

- **需求描述**：无需手动开机敲命令拉起服务，且需记录详尽的下载与操作日志以便排查问题。
- **系统自启动**：
  - *Windows 环境*：将 Nginx 和后台 Python 服务封装为 Windows Service，或编写 `.bat` 脚本加入“任务计划程序”（设置开机无登录自启）。
  - *Linux 环境*：编写 `systemd` 服务配置文件（`cloud-drive.service`），实现 `systemctl enable --now` 自启动及崩溃自动重启。
- **日志记录规范**：
  - 引入统一日志模块，日志输出格式：`[时间戳] [客户端公网IPv6] [操作类型: UPLOAD/DOWNLOAD/DELETE] [文件路径/名称] [状态码] [文件大小] [耗时]`。
  - 日志文件按天进行滚动切分（Log Rotation），本地默认保留最近 30 天的记录。

## 5. 拓展功能需求 (高级阶段)

### 5.1 前端文件管理系统 (Web GUI)

- **需求描述**：摆脱 `uploadserver` 简陋的默认界面，为管理员和用户提供类似于正规网盘的交互视图。
- **功能要点**：
  - **目录树浏览**：支持文件夹层级展现，支持双击进入下一级目录。
  - **常规操作**：在前端界面直接进行【新建文件夹】、【重命名】、【单文件/多文件删除】、【移动/剪切】。
  - **多选与批量**：支持批量删除和批量下载（自动在后端打包为 `.zip` 后下载）。
  - **方案升级建议**：建议直接采用或参考开源成熟的本地文件管理工具（如 `File Browser` ），它自带美观的 Web UI、完善的用户权限控制、日志系统和 HTTPS 支持，能极大减少重复造轮子的时间。

### 5.2 AI 能力拓展（呼应“AI个人云盘”定位）

*为了让您的项目名称“AI个人云盘”名副其实，建议在后期分阶段引入以下轻量级 AI 特性：*

- **特性一：AI 智能文件分类与标签（轻量级视觉/文本模型）**
  - **功能**：文件上传后，后台自动调用本地轻量级 AI 模型。
  - **效果**：图片上传后自动识别物体或场景并打上标签（如“发票”、“风景”、“实验截图”）；文档上传后自动识别类别（如“论文”、“账单”、“代码”），方便后续分类检索。
- **特性二：基于 RAG 的“文档本地大模型对话”**
  - **功能**：在本地通过 Ollama 运行轻量级大模型（如 Qwen-1.5B 或 Llama3-8B）。
  - **效果**：用户可以直接在云盘 Web 界面勾选上传的 PDF 论文、教材或资料，输入问题，AI 自动阅读云盘文件并给出摘要、翻译或核心内容解答。
- **特性三：语义化搜索 (Semantic Search)**
  - **功能**：引入轻量级向量数据库（如 Chroma）。
  - **效果**：搜索文件不再局限于死板的“文件名精确匹配”。输入“去年的旅游照片”或“关于深度学习的笔记”，AI 能够通过语义理解直接召回相关文件。

## 6. 非功能性需求与潜在风险提示

1. **网络稳定性依赖**：由于依赖手机热点作为网关，若手机离开电脑、热点断开或处于无信号地带，外网将无法访问云盘。
2. **安全防护（重中之重）**：
   - 一旦暴露在公网 IPv6 中，极易遭受扫描器的自动化攻击。
   - **必须在文件管理页面前置身份验证（强密码登录机制）**。
   - 严格限制上传的文件类型（严禁上传并直接执行 `.exe`、`.sh`、`.bat` 等可执行文件，防止被注入木马）。
